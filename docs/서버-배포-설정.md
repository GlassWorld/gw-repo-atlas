# 서버 배포 설정

이 문서는 `atlas.glassworld.co.kr`를 같은 서버에서 `home.glassworld.co.kr`와 함께 운영하기 위해 실제로 적용한 설정 절차를 정리합니다.

## 현재 운영 구조

- `home.glassworld.co.kr`는 `/home/ubuntu/app/frontend`의 Docker Compose로 운영합니다.
- 외부 `80`, `443` 포트는 `gw-fe-nginx` 컨테이너가 점유합니다.
- `atlas.glassworld.co.kr`는 `/home/ubuntu/app/atlas`의 Docker Compose로 운영합니다.
- Atlas 앱 컨테이너 이름은 `repo-atlas-app`입니다.
- Atlas 앱은 서버에서 `3001` 포트로 서비스합니다.
- `gw-fe-nginx`가 `atlas.glassworld.co.kr` 요청을 Atlas 앱으로 프록시합니다.

## 경로 정리

### Home 프론트 운영 경로

- `/home/ubuntu/app/frontend/docker-compose.yml`
- `/home/ubuntu/app/frontend/nginx.conf`
- `/home/ubuntu/app/frontend/certbot/conf/live/home.glassworld.co.kr/`
- `/home/ubuntu/app/frontend/certbot/conf/live/atlas.glassworld.co.kr/`

### Atlas 운영 경로

- `/home/ubuntu/app/atlas/docker-compose.yml`
- `/home/ubuntu/app/atlas/.env`

## DNS

- 가비아에서 `atlas.glassworld.co.kr`를 운영 서버 IP로 연결합니다.
- DNS 연결만으로 HTTPS가 되지는 않습니다.
- HTTPS는 `gw-fe-nginx` 컨테이너의 인증서와 Nginx 설정이 처리합니다.

## 인증서

Atlas 인증서는 `home`과 같은 Certbot 볼륨 구조를 사용합니다.

인증서 경로:

- `/home/ubuntu/app/frontend/certbot/conf/live/atlas.glassworld.co.kr/fullchain.pem`
- `/home/ubuntu/app/frontend/certbot/conf/live/atlas.glassworld.co.kr/privkey.pem`

컨테이너 내부에서 참조되는 경로:

- `/etc/letsencrypt/live/atlas.glassworld.co.kr/fullchain.pem`
- `/etc/letsencrypt/live/atlas.glassworld.co.kr/privkey.pem`

인증서 파일 확인:

```bash
ls -la /home/ubuntu/app/frontend/certbot/conf/live/atlas.glassworld.co.kr/
```

## Home Nginx 설정

`/home/ubuntu/app/frontend/nginx.conf` 안의 `http {}` 블록 안에 Atlas용 `server` 블록을 추가합니다.

```nginx
server {
  listen 80;
  server_name atlas.glassworld.co.kr;
  client_max_body_size 20M;

  location /.well-known/acme-challenge/ {
    root /var/www/certbot;
  }

  location / {
    return 301 https://$host$request_uri;
  }
}

server {
  listen 443 ssl;
  server_name atlas.glassworld.co.kr;
  client_max_body_size 20M;

  ssl_certificate /etc/letsencrypt/live/atlas.glassworld.co.kr/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/atlas.glassworld.co.kr/privkey.pem;
  ssl_protocols TLSv1.2 TLSv1.3;
  ssl_session_timeout 1d;
  ssl_session_cache shared:SSL:10m;
  ssl_session_tickets off;

  gzip on;
  gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;
  gzip_min_length 1024;

  location / {
    proxy_pass http://host.docker.internal:3001;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";

    proxy_connect_timeout 10s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;
  }
}
```

주의:

- `server` 블록은 반드시 `http {}` 안에 있어야 합니다.
- `proxy_pass http://$atlas_upstream;`처럼 변수 경유 방식은 피하고, `proxy_pass http://host.docker.internal:3001;`를 직접 사용합니다.

## Home Docker Compose 설정

`gw-fe-nginx` 컨테이너가 호스트의 Atlas 앱으로 접근할 수 있도록 `/home/ubuntu/app/frontend/docker-compose.yml`의 `nginx` 서비스에 `extra_hosts`를 추가합니다.

```yaml
services:
  nginx:
    image: nginx:latest
    container_name: gw-fe-nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

적용:

```bash
cd /home/ubuntu/app/frontend
docker compose up -d --force-recreate nginx
```

## Atlas 앱 설정

Atlas 앱은 `/home/ubuntu/app/atlas`에서 별도 Compose로 실행합니다.

권장 `.env` 값:

```env
APP_PORT=3001
APP_BIND_IP=0.0.0.0
DATABASE_URL=...
OPENAI_API_KEY=...
AUTH_JWT_SECRET=...
```

중요:

- `APP_PORT`는 `3001`로 통일합니다.
- `APP_BIND_IP=127.0.0.1`이면 `gw-fe-nginx` 컨테이너에서 접근하지 못해 `502 Bad Gateway`가 발생할 수 있습니다.
- 같은 서버 내 다른 컨테이너에서 접근해야 하므로 `APP_BIND_IP=0.0.0.0`을 사용합니다.

Atlas 재기동:

```bash
cd /home/ubuntu/app/atlas
docker compose down
docker compose up -d
```

## 점검 명령

### Atlas 앱 직접 확인

```bash
curl -I http://127.0.0.1:3001
docker ps
docker logs --tail 100 repo-atlas-app
```

정상일 경우 `curl -I http://127.0.0.1:3001`는 `302 Found`와 `/login` 리다이렉트를 반환할 수 있습니다.

### Nginx 컨테이너에서 Atlas 연결 확인

```bash
docker exec gw-fe-nginx curl -I http://host.docker.internal:3001
```

정상일 경우 `302 Found` 또는 다른 정상 HTTP 응답이 반환됩니다.

### 최종 HTTPS 확인

```bash
curl -Ik https://atlas.glassworld.co.kr
```

정상일 경우 `502`가 아니라 `200`, `302` 등의 응답이 반환됩니다.

## 자주 발생한 문제

### 1. HTTPS가 안 붙는 경우

- 원인: Atlas용 서버 블록이 `gw-fe-nginx` 설정에 없음
- 원인: Atlas 인증서 경로를 잘못 참조함
- 원인: `home`용 Docker nginx가 외부 `443`을 이미 점유하고 있는데 호스트 nginx를 수정함

### 2. `502 Bad Gateway`

- 원인: `repo-atlas-app`이 떠 있지 않음
- 원인: Atlas 앱이 `3001`이 아닌 다른 포트로 뜸
- 원인: `APP_BIND_IP=127.0.0.1`이라 Nginx 컨테이너가 접근하지 못함
- 원인: `proxy_pass`를 변수 방식으로 작성해 `host.docker.internal` 해석이 꼬임

### 3. 인증서는 있는데 HTTPS가 안 되는 경우

- 원인: 인증서 파일은 존재하지만 `gw-fe-nginx`가 그 도메인에 대한 `server_name` 블록을 갖고 있지 않음
- 원인: `nginx.conf`는 수정했지만 컨테이너를 강제 재생성하지 않아 구 버전 설정이 유지됨

## 최종 반영 순서

1. 가비아 DNS에서 `atlas.glassworld.co.kr`를 서버 IP로 연결합니다.
2. Atlas 인증서를 Certbot 경로에 준비합니다.
3. `/home/ubuntu/app/frontend/nginx.conf`에 Atlas `server` 블록을 추가합니다.
4. `/home/ubuntu/app/frontend/docker-compose.yml`에 `extra_hosts`를 추가합니다.
5. `docker compose up -d --force-recreate nginx`로 Home Nginx 컨테이너를 다시 만듭니다.
6. `/home/ubuntu/app/atlas/.env`에서 `APP_PORT=3001`, `APP_BIND_IP=0.0.0.0`을 확인합니다.
7. `/home/ubuntu/app/atlas`에서 Atlas 앱을 재기동합니다.
8. `curl -I http://127.0.0.1:3001`와 `curl -Ik https://atlas.glassworld.co.kr`로 최종 확인합니다.
