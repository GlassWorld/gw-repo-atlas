# Repo-Atlas 구조 개요

## 프로젝트 목적

Repo-Atlas는 GitHub 저장소를 분석하여 개발자가 낯선 코드베이스를 빠르게 이해할 수 있도록 돕는 웹 애플리케이션입니다.

## 주요 디렉터리

- `pages/`: 화면 단위 페이지
- `components/`: 재사용 가능한 UI 컴포넌트
- `server/api/`: API 엔드포인트
- `server/services/`: 핵심 비즈니스 로직
- `server/db/`: Prisma 클라이언트
- `server/utils/`: 공통 유틸리티
- `prisma/`: 데이터베이스 스키마와 마이그레이션
- `.ai/`: AI 에이전트용 영문 작업 문서
- `docs/`: 사람용 한글 문서

## 핵심 흐름

1. 사용자가 GitHub 저장소 URL을 입력합니다.
2. 서버가 저장소를 등록하고 분석 작업을 생성합니다.
3. Git clone으로 파일 구조와 커밋을 수집합니다.
4. OpenAI를 이용해 프로젝트 요약과 읽기 순서를 생성합니다.
5. 사용자는 분석 결과와 Q&A 화면에서 저장소를 탐색합니다.
