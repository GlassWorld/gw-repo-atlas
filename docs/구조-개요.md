# Repo-Atlas 구조 개요

## 프로젝트 목적

Repo-Atlas는 GitHub 저장소를 분석하여 개발자가 낯선 코드베이스를 빠르게 이해할 수 있도록 돕는 웹 애플리케이션입니다.

## 주요 디렉터리

- `pages/`: 화면 단위 페이지
- `components/`: 재사용 가능한 UI 컴포넌트
- `server/api/`: API 엔드포인트
- `server/services/`: 핵심 비즈니스 로직
- `server/db/`: Prisma 클라이언트
- `server/utils/`: 공통 유틸리티
- `utils/`: 화면과 서버가 함께 쓰는 분석 카탈로그와 표시 유틸리티
- `prisma/`: 데이터베이스 스키마와 마이그레이션
- `.ai/`: AI 에이전트용 영문 작업 문서
- `docs/`: 사람용 한글 문서

## 핵심 흐름

1. 사용자가 GitHub 저장소 URL을 입력합니다.
2. 서버가 저장소를 등록하고 분석 작업을 생성합니다.
3. Git clone으로 파일 구조와 커밋을 수집합니다.
4. 파일트리와 스니펫으로 프로젝트 종류를 감지합니다.
5. OpenAI를 이용해 프로젝트 요약과 스택별 헬스 스코어를 생성합니다.
6. 사용자가 필요한 분석 항목을 실행하면 항목별 상세 리포트를 별도로 생성합니다.
7. 사용자는 분석 결과와 Q&A 화면에서 저장소를 탐색합니다.

## 분석 구조

기본 분석은 저장소 단위로 최신 결과를 재사용합니다. 다시 분석을 실행하면 새 이력 행을 계속 쌓기보다 같은 저장소의 최신 분석을 재실행합니다.

항목별 분석은 `analysis_artifact`에 저장됩니다. 현재 항목 카탈로그는 `utils/analysis-items.ts`에서 관리하며, 화면 목록과 API 검증, 상세 페이지 제목이 같은 정의를 사용합니다.

비동기 실행은 `analysis_job` 큐를 사용합니다. API는 작업을 등록하고, Nitro 서버 플러그인의 워커가 대기 중인 작업을 가져와 기본 분석 또는 항목 분석을 실행합니다.

## 표시 구조

분석 상태와 헬스 점수의 색상·이모지 기준은 `utils/analysis-presentation.ts`에 둡니다. 페이지와 패널이 같은 기준을 공유하므로 완료, 진행, 실패, 대기 상태가 화면 전반에서 일관되게 표시됩니다.
